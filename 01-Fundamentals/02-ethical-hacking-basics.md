# Ethical Hacking Basics

Ethical hacking (white-hat hacking) is the authorized practice of bypassing system security to identify potential data breaches and threats in a network.

## Core Hacking Phases
1. **Reconnaissance** - Passive and active information gathering
2. **Scanning & Enumeration** - Live hosts, open ports, services
3. **Gaining Access** - Exploit discovered vulnerabilities
4. **Privilege Escalation** - Move from low to high privilege
5. **Reporting** - Document findings with PoC and remediation

## Key Commandments for Bug Hunters
1. Stay strictly in scope
2. Do no harm - avoid destructive payloads, DoS
3. Minimize data access - only access what's needed to prove the bug
4. Report clearly - write actionable, reproducible reports
5. Never publicly disclose without permission
6. Respect program rules

## Required Mindset
- **Curiosity** - Question everything
- **Persistence** - Most attempts fail; keep testing
- **Humility** - Everyone misses things; learn from triage feedback
- **Ethics** - Your reputation is your most valuable asset
