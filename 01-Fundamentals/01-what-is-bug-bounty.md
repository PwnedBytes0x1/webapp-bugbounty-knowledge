# What is Bug Bounty?

Bug bounty programs are crowd-sourced security initiatives where organizations invite independent security researchers to find and report vulnerabilities in their applications, infrastructure, or systems. Participants are rewarded based on the severity of the bug they discover.

## How It Works
1. **Scope Definition** - The organization defines which assets are in scope
2. **Researcher Engagement** - Researchers analyze the target for security weaknesses
3. **Report Submission** - Valid findings are submitted through a platform
4. **Triage & Validation** - The security team validates and assesses severity
5. **Reward & Disclosure** - Researcher receives bounty upon validation

## Types of Programs
| Type | Description |
|------|-------------|
| Public | Open to all researchers |
| Private/Invite-only | By invitation, less competition |
| VDP | No monetary reward, recognition only |
| Live Hacking Events | Time-bound events with bonuses |

## Common Platforms
- **HackerOne** - Google, Twitter, GitHub, Yahoo, US DoD
- **Bugcrowd** - Tesla, Amazon, Atlassian, Spotify
- **Intigriti** - European-based, KPN, Sony
- **YesWeHack** - French platform with strong EU coverage
- **Synack** - Invite-only, paid hourly rates

## Skills Required
- HTTP, DOM, JavaScript, APIs
- OWASP Top 10 understanding
- Burp Suite proficiency
- Reconnaissance methodology
- Report writing
- Persistence and patience
