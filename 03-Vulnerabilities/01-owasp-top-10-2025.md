# OWASP Top 10 2025: Complete Breakdown

## A01:2025 — Broken Access Control

### Detection
- Test horizontal privilege escalation (User A accesses User B's data)
- Test vertical privilege escalation (User accesses Admin functions)
- Test parameter manipulation (IDs, roles, permissions)
- Test HTTP method override (POST->PUT->DELETE on protected endpoints)
- Test missing access controls on API endpoints discovered via JS/Archive

### Bypass Techniques
- UUID enumeration via batch/alias queries (GraphQL)
- Insecure Direct Object References (IDOR) in JSON arrays
- Mass assignment in REST APIs (extra fields in JSON body)
- Path traversal in URL-based access decisions
- Cookie/header manipulation for role impersonation

## A02:2025 — Cryptographic Failures

### Detection
- Weak TLS versions (< 1.2)
- Missing HSTS headers
- Hardcoded secrets in source code / JS files
- Predictable session tokens / JWTs
- Weak password hashing (MD5, SHA1 for passwords)
- Stored credit card numbers without encryption

### Exploitation
- JWT signature verification disabled (alg: none)
- JWT algorithm confusion (RS256 -> HS256 with public key)
- Weak JWT secret brute force
- Predictable CSRF tokens
- ECB ciphertext manipulation (cut-and-paste attack)
- Padding oracle attacks

## A03:2025 — Injection

### Detection
- Entry points: all user-controlled input reaching backends
- Type: SQL, NoSQL, OS command, LDAP, XPath, Template (SSTI)
- Character reflection: special chars produce errors
- Timing: response delay reveals time-based injection

### Full Coverage
See individual files: SQLi, NoSQLi, SSTI, Command Injection, XXE

## A04:2025 — Insecure Design

### Detection
- Business logic flaws in multi-step workflows
- Missing rate limiting (brute force, OTP bypass)
- Inconsistent security rules between API versions
- Race conditions in financial/state-changing operations
- Trusting client-side validation

## A05:2025 — Security Misconfiguration

### Detection
- Default credentials unchanged
- Debug/error pages enabled (stack traces, verbose errors)
- Directory listing enabled
- Unnecessary open ports/services
- Cloud storage (S3) publicly accessible
- Missing security headers (CSP, HSTS, X-Frame-Options)
- Unpatched software versions

## A06:2025 — Vulnerable & Outdated Components

### Detection
- Version disclosure in headers (X-Powered-By, Server)
- Known CVEs for identified versions
- JS library versions (check for known vulns)
- Third-party CDN scripts without SRI
- Orphaned microservices still running old code

## A07:2025 — Identification & Authentication Failures

### Detection
- No account lockout / rate limiting on login
- Weak password policy
- Predictable password reset tokens
- No MFA enforcement on sensitive actions
- Session fixation
- JWT not invalidated on logout
- OAuth misconfiguration (account linking CSRF)

## A08:2025 — Software & Data Integrity Failures

### Detection
- Unsigned software updates (man-in-the-middle)
- CI/CD pipeline without integrity verification
- Dependency confusion (npm, pip, gem, maven)
- Insecure deserialization

## A09:2025 — Security Logging & Monitoring Failures

### Detection
- No logging on sensitive operations (password changes, privilege escalation)
- Log injection via input (CRLF in logs)
- Monitoring gaps for known attack patterns

## A10:2025 — Server-Side Request Forgery (SSRF)

See ssrf.md for full coverage.

## Key Changes from 2021 to 2025
- SSRF promoted from "should be in Top 10" to A10 standalone
- A04 Insecure Design expanded to include AI/ML pipeline flaws
- A08 now covers software supply chain integrity
- Deserialization moved under A08
- A06 now includes cloud-native component vulnerabilities
